# Security Policy

## Supported Versions

We release security updates for the following versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take security vulnerabilities seriously. If you discover a security issue, please follow these steps:

### Do NOT:
- ❌ Open a public GitHub issue
- ❌ Discuss the vulnerability publicly
- ❌ Submit a PR with the fix immediately

### DO:
1. ✅ **Email us directly** at [security@kodaskills.co](mailto:security@kodaskills.co?subject=[VULNERABILITY]&body=) with details
2. ✅ **Include**:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if you have one)
3. ✅ **Allow response time**: We'll acknowledge within 48 hours and provide updates on our progress

## Response Process

1. **Acknowledgment**: Within 48 hours
2. **Investigation**: We'll verify and assess the issue
3. **Fix Development**: We'll work on a patch
4. **Release**: Security update published
5. **Disclosure**: Public disclosure after fix is available

## Security Best Practices for Users

- Always use the latest version
- Review dependency updates before upgrading
- Report security concerns promptly and privately

## Security Best Practices for Contributors

- Never commit secrets, API keys, or credentials
- Use environment variables for sensitive configuration
- Follow secure coding practices
- Review dependencies for known vulnerabilities
